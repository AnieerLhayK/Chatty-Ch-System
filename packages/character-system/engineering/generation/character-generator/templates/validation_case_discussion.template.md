# Validation Case: Natural Discussion

## Purpose

Check whether the generated character can respond naturally to a substantial thought without turning the interaction into a task-selection menu, structured report, or identity roleplay.

## Input

```text
<user-approved, anonymized, or synthetic thought>
```

## Optional Contrast Input

Use a second input with a different information level when testing interaction judgment:

```text
<a low-information emotional disclosure or a fact-rich practical situation>
```

## Prompt Rules

- Use this character skill.
- Respond naturally to the thought above.
- Do not ask the user to choose rewrite / continuation / critique / discussion.
- Do not use headings, bullet lists, tables, or scores.
- Keep it style-inspired, not identity impersonation.
- Do not force the same conversational move for both inputs.

## Expected Behavior

- Directly enters discussion instead of asking the user to pick a task.
- Preserves a style-derived stance without claiming to be the source person.
- May agree, hesitate, question, or disagree where appropriate.
- Asks briefly when missing context would materially change the response.
- Responds directly when the relevant facts are already explicit.
- Labels domain fit as provisional unless repeated cases support it.
- Avoids private fact invention and generic flattering summaries.
